# Table: order_items

## Business Purpose

The `order_items` table stores the individual products (line items) included in each customer order. Each record represents a single product purchased within an order, along with its seller, selling price, shipping deadline, and freight cost.

This table acts as the transactional detail layer of the Olist marketplace and connects customer orders with products and sellers. It enables detailed sales, pricing, seller performance, and logistics analysis.

---

## Business Process

The business process represented by this table is:

```text
Customer Places Order
          │
          ▼
Order Created
          │
          ▼
Products Added to Order
          │
          ▼
Seller Assigned
          │
          ▼
Shipping Deadline Defined
          │
          ▼
Item Shipped
```

An order may contain one or more products.

Each product purchased creates one record in the `order_items` table.

---

## Grain

**One row represents one product (line item) within one customer order.**

This is a lower level of detail than the `orders` table, which stores one record per order.

---

## Primary Key

**Composite Primary Key**

* `order_id`
* `order_item_id`

The combination of these two columns uniquely identifies each line item within an order.

---

## Foreign Keys

| Foreign Key  | References            | Purpose                                                    |
| ------------ | --------------------- | ---------------------------------------------------------- |
| `order_id`   | `orders.order_id`     | Identifies the order containing the item.                  |
| `product_id` | `products.product_id` | Identifies the purchased product.                          |
| `seller_id`  | `sellers.seller_id`   | Identifies the seller responsible for fulfilling the item. |

---

## Table Relationships

```text
Customers
     │
     ▼
 Orders
     │
     ▼
Order Items
   │      │
   ▼      ▼
Products Sellers
```

The `order_items` table serves as the bridge between orders, products, and sellers.

---

## Column Descriptions

| Column                | Description                                                                        |
| --------------------- | ---------------------------------------------------------------------------------- |
| `order_id`            | Unique identifier of the order containing the item.                                |
| `order_item_id`       | Sequential number identifying the item within the order.                           |
| `product_id`          | Unique identifier of the purchased product.                                        |
| `seller_id`           | Unique identifier of the seller responsible for the item.                          |
| `shipping_limit_date` | Deadline by which the seller should dispatch the product to the logistics carrier. |
| `price`               | Selling price of the individual product.                                           |
| `freight_value`       | Shipping (freight) cost associated with the item.                                  |

## Data Quality Checks

Before importing the data into PostgreSQL, perform the following validations:

### Primary Key Validation

* Verify that the combination of (`order_id`, `order_item_id`) is unique.
* Check for missing values in either key column.

### Foreign Key Validation

* Verify every `order_id` exists in the `orders` table.
* Verify every `product_id` exists in the `products` table.
* Verify every `seller_id` exists in the `sellers` table.

### Missing Values

* Check for missing product identifiers.
* Check for missing seller identifiers.
* Check for missing shipping deadlines.
* Check for missing price or freight values.

### Numeric Validation

* Ensure `price` is greater than or equal to zero.
* Ensure `freight_value` is greater than or equal to zero.
* Identify unusually high prices or freight costs for further investigation.

### Duplicate Records

* Verify there are no duplicate (`order_id`, `order_item_id`) combinations.

## Business Importance

The `order_items` table is one of the most valuable transactional tables in the project because it provides product-level sales information. While the `orders` table tells us that an order occurred, the `order_items` table tells us exactly **what was purchased, who sold it, how much it cost, and how much shipping was charged**.

This table is essential for:

* Sales analysis
* Product performance analysis
* Seller performance evaluation
* Revenue reporting
* Freight cost analysis
* Product category analysis
* Executive dashboards

Many key business metrics—including total revenue, product sales, seller rankings, and category performance—are derived directly from this table.

---

## Notes & Assumptions

* One order can contain multiple products.
* The same product may appear in many different orders.
* Different products within the same order may be sold by different sellers.
* The `price` column represents the selling price of a single item, **not** the total order value.
* The `freight_value` represents the shipping cost associated with the individual item.
* Revenue calculations should be performed using the `price` column from this table rather than the `orders` table.
* The composite primary key (`order_id`, `order_item_id`) should be validated during database design.
